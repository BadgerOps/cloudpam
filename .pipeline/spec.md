# Goal

Replace the embedded static `docs/openapi.yaml` serving path with a live OpenAPI generator that is derived from the routes registered in the running server. Remove the maintained static snapshot and make `/openapi.yaml` reflect runtime configuration, such as whether metrics were registered.

# Non-Goals

- Do not refactor handler behavior or change endpoint semantics.
- Do not replace the HTTP router.
- Do not attempt full request/response examples for every endpoint in this pass.

# User-Facing Behavior

- `GET /openapi.yaml` returns an OpenAPI 3.1 YAML document generated by the running binary.
- `GET /openapi` returns an interactive Scalar API reference backed by `/openapi.yaml`.
- The generated spec includes current registered API paths, HTTP methods, auth requirements, tags, summaries, query/path parameters, request body schema references, and response schema references where known.
- `docs/openapi.yaml` is removed and is no longer a maintained artifact.
- Validation commands use the Nix development environment.

# Technical Approach

- Add a small route registry to `internal/api` and route registration wrappers that both register with `http.ServeMux` and record the mux pattern.
- Generate OpenAPI from recorded route patterns plus a code-side operation catalog keyed by method/path.
- Use reflection over Go structs to produce component schemas from JSON tags, so model shape stays closer to actual domain/auth types.
- Replace `docs/spec_embed.go` use for OpenAPI serving; keep changelog embedding.
- Replace Ruby validation with Go validation and update `Justfile` docs/tooling around `/openapi.yaml`.
- Add a concrete `0.14.6` changelog entry; do not use an `Unreleased` section.
- Add focused tests for generated spec contents and runtime configuration behavior.

# Edge Cases

- Prefix mux registrations such as `/api/v1/pools/` need expansion to concrete documented paths.
- Method-qualified Go 1.22 mux patterns such as `POST /api/v1/import/accounts` should preserve the method.
- Metrics is only present when metrics are configured.
- Static SPA catch-all `/` should not appear in the OpenAPI document.
- Duplicate operation entries should be deduplicated deterministically.

Issue Types & Resolution Actions
What each issue type means and how to resolve them
⚠ Drift
Subnet exists in both IPAM and cloud, but properties differ (name, AZ, tags, etc.)
Resolution Options:
  • Accept Cloud State - update IPAM to match
  • Update Cloud - push IPAM values to cloud
  • Dismiss - ignore this difference
? Unmanaged
Subnet found in cloud but not tracked in CloudPAM. May be manually created or from another tool.
Resolution Options:
  • Import - add to CloudPAM tracking
  • Assign to Pool - import under a parent
  • Ignore - exclude from future scans
✗ Missing
Subnet tracked in IPAM but not found in cloud. May have been deleted outside IPAM.
Resolution Options:
  • Remove from IPAM - clean up stale record
  • Recreate in Cloud - provision again
  • Keep in IPAM - retain for documentation

Implementation Notes

  • Sync runs on configurable schedule (default: 15 min) or on-demand
  • Each account can have different sync settings
  • Bulk actions allow resolving multiple issues at once
  • All resolutions are logged in audit trail
  • Webhooks can notify on new drift detection
